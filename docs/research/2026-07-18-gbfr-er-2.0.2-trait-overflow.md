# Granblue Fantasy: Relink 2.0.2 장비 특성 초과 조사

작성일: 2026-07-18
대상: Granblue Fantasy: Relink - Endless Ragnarok 2.0.2, Windows x64

## 1. 조사 목적

로컬 캐릭터가 장착한 진과 장비에서 같은 특성(Trait)이 중복될 때, 합산 레벨이 최대 유효 레벨을 넘는지 MOD에서 안전하게 판정할 수 있는지 확인한다. 여기서 “스킬”은 캐릭터의 공격 스킬이 아니라 `대미지 상한`, `공격력`, `퀵 어빌리티` 같은 장비 특성을 뜻한다.

## 2. 결론

구현 가능하다. 구버전 공개 훅에는 로컬 플레이어의 장착 진 12개, 각 진의 주·보조 특성 ID와 레벨, 무기 특성, 가호석, 오버마스터리와 스탯을 읽어 전송하는 구현이 있었다. 현재 Djeeta MOD에도 해당 프로토콜 타입과 파서·로그 UI가 남아 있다.

다만 Endless Ragnarok 2.0.2에서는 플레이어와 장비 메모리 레이아웃이 이동했다. 현재 훅은 검증되지 않은 장비 데이터를 제조하지 않기 위해 장비 읽기를 비활성화하고 플레이어 신원만 전송한다. 따라서 2.0.2 전용 장비 구조와 특성 최대치 테이블을 다시 검증해야 한다.

최대치 판정에는 적, 방어력, 속성, 실제 대미지 계산이 필요하지 않다. 필요한 값은 다음 두 가지뿐이다.

- 선택한 로컬 캐릭터에게 적용되는 `trait ID → 합산 레벨`
- 2.0.2 기준 `trait ID → 최대 유효 레벨`

예를 들어 대미지 상한 합계가 72이고 최대 유효 레벨이 65라면 `72/65 · 7 초과`로 표시할 수 있다.

## 3. 2.0.2와 DLC에서 확인된 관련 변경

Cygames는 Endless Ragnarok 출시와 함께 2.0.2를 배포했다. 확장팩에는 Master Traits, Summons, 신규 진, 무기 Transcendence가 추가되었다. 따라서 1.3.x 시점의 진 표만으로 2.0.2 전체 장비 효과를 확정하면 안 된다.

공식 2.0.2 패치 노트에서 확인되는 관련 사항은 다음과 같다.

- 신규 진이 Chaos 이상 퀘스트 보상으로 추가되었다.
- 대미지 상한 특성 효과가 PWR에 반영된다.
- 같은 특성을 가진 진을 함께 장착했을 때 총 특성 레벨이 잘못 표시되던 문제가 수정되었다.
- 진과 가호석을 희귀도 또는 특성으로 필터링할 수 있게 되었다.
- 추가 특성이 있는 진 아이콘에 `+`가 표시된다.
- Master Traits와 Summons라는 별도 성장·장착 체계가 추가되었다.

이 변경은 게임 자체가 캐릭터별 특성 합계를 계산하고 있음을 뒷받침한다. MOD는 그 UI를 조작하지 않고, 동일한 원천 데이터를 읽어 독립적으로 합산하는 방식을 사용한다.

## 4. 특성과 최대 유효 레벨

비공식 Relink 위키의 1.3.x 데이터에서 `DMG Cap` 특성은 최대 T.Lvl 65로 기록되어 있다. 그러나 이 수치를 포함한 모든 최대치는 2.0.2 게임 데이터에서 다시 추출하고 검증해야 한다.

공개 모딩 문서에 따르면 캐릭터의 action/parameter 데이터에는 공격별 대미지 상한 배열과 참조 인덱스가 존재한다. 이것은 공격 대미지 상한 계산에는 유용하지만, 이번 기능이 다루는 장비 특성 최대 레벨과는 별개의 데이터다. 이번 기능은 공격별 cap 수치가 아니라 장비 화면에서 합산되는 trait level과 trait level cap만 사용한다.

최대치 카탈로그는 다음 원칙으로 만든다.

- 2.0.2 자산에서 추출한 trait ID를 키로 사용한다.
- 최대 유효 레벨이 확인된 특성만 `정상/최대/초과`로 판정한다.
- 신규·캐릭터 고유 특성처럼 최대치가 확인되지 않은 항목은 `최대치 미확인`으로 표시한다.
- 추출에 사용한 게임 실행 파일 또는 관련 자산의 SHA-256을 문서화한다.
- 구버전 위키 값은 교차 검증 자료로만 사용한다.

## 5. 공개 훅과 현재 저장소의 근거

False Spring의 구버전 `gbfr-logs` 플레이어 로드 훅은 다음 값을 읽었다.

- 진 12칸
- 각 진의 `first_trait_id`, `first_trait_level`
- 각 진의 `second_trait_id`, `second_trait_level`
- 진 ID, 진 레벨, 장착 캐릭터
- 무기의 세 특성 ID와 레벨
- 가호석 ID
- 오버마스터리와 캐릭터 스탯

Djeeta MOD의 `protocol/src/lib.rs`에도 `Sigil`, `WeaponInfo`, `OvermasteryInfo`, `PlayerStats`, `PlayerLoadEvent`가 남아 있다. Tauri 파서는 `PlayerLoadEvent`를 완전한 `PlayerData`로 변환하며, 저장 로그의 Equipment 화면도 진과 무기를 렌더링할 수 있다.

현재 `src-hook/src/hooks/player.rs`는 2.0.2에서 검증된 플레이어 신원 스냅샷만 읽는다. `src-hook/src/hooks/mod.rs`에는 장비 레이아웃이 확인될 때까지 equipment capture를 비활성화한다는 의도가 명시돼 있다. 따라서 구버전 오프셋을 그대로 되살리는 것은 안전하지 않다.

## 6. 2.0.2 구현 시 필요한 역공학

1. 로컬 캐릭터 목록과 각 캐릭터의 장비 레코드를 찾는다.
2. 진 12칸, 주·보조 특성, 무기 특성, 가호석 특성의 2.0.2 레이아웃을 검증한다.
3. Master Traits와 Summon traits가 일반 trait level 합계에 참여하는지 확인한다.
4. 참여한다면 동일한 trait ID/level 입력으로 수집한다.
5. 어느 필수 원천이든 확인되지 않으면 완전한 합계를 표시하지 않고 `장비 정보 미지원`으로 처리한다.
6. MOD 계산 결과를 게임 장비 화면의 총 특성 레벨과 대조한다.

포인터와 레이아웃은 다음 조건을 모두 통과해야 한다.

- 지원 실행 파일 또는 자산 해시 일치
- 검색 시그니처의 유일한 매치
- 캐릭터 ID와 슬롯 인덱스의 유효 범위
- 진 슬롯 수 12 이하
- 특성 ID와 레벨의 합리적인 범위
- 같은 캐릭터의 반복 스냅샷이 결정적으로 동일함

실패한 읽기는 빈 장비나 0레벨로 바꾸지 않는다. 잘못된 합계를 정상 결과처럼 보여주는 것보다 명시적인 미지원 상태가 안전하다.

## 7. 표시 범위

첫 구현은 장비 메뉴 선택 감지에 의존하지 않는다. MOD 프로그램의 별도 `장비 분석` 화면에서 모든 로컬 캐릭터를 선택할 수 있게 한다.

특성은 다음 순서로 표시한다.

1. 최대치를 초과한 특성
2. 최대치에 도달한 특성
3. 최대치 미만인 특성
4. 최대치가 확인되지 않은 특성

각 행은 한국어 특성명, 현재 레벨, 최대 유효 레벨, 초과 레벨을 표시한다. 상세 보기에는 어떤 진·무기·가호석 또는 2.0.2 추가 장비 효과가 몇 레벨을 제공했는지 표시한다.

## 8. 위험과 제한

- 모든 메모리 오프셋과 시그니처는 2.0.2 전용이며 업데이트 후 재검증이 필요하다.
- Master Traits와 Summons가 기존 trait level과 다른 효과 체계를 사용하면 일반 특성 합계에 섞지 않는다.
- 온라인 파티원의 장비는 범위 밖이다.
- 전투 중 버프·디버프, 적 방어력, 공격별 실제 대미지 상한은 범위 밖이다.
- 게임 저장 데이터나 장비를 변경하지 않는다. 읽기 전용으로만 동작한다.
- 수동 2.0.2 스모크 테스트 전에는 호환성을 주장하지 않는다.

## 9. 출처

### 2.0.2 특성 상한 카탈로그 출처

- 대상 게임 실행 파일: `granblue_fantasy_relink.exe`
- 실행 파일 SHA-256: `63340832BCF731FBC97796F686B05C988418E83D451D4A49B2244A85D00E297F`
- 추출 도구: `Nenkai/GBFRDataTools` 태그 `2.0.0`
- 도구 배포 ZIP SHA-256: `2F355E7785D7ED7D1A4F99B1FCCC626BB9D949CE29A4F08B816A233DAB77F63B`
- 원본 테이블: `system/table/skill_status.tbl`
- 원본 테이블 SHA-256: `96D56E65F107FD925B131D86959C9F829CE7102E6BDD39C7C6F3E80F663E7563`
- 생성 결과: 고유 특성 261개, `src-tauri/assets/trait-caps.json`
- 생성 결과 SHA-256: `087A508B50EE6A9E926AD69B870C5FE11EDFACE556396C0111F77DA3BB3843AC`

`skill_status.Key`는 `SKILL_020_00` 같은 심볼 키와 이미 해시화된 8자리 키가 함께 존재한다. 심볼 키에는 GBFRDataTools 2.0.0의 `XXHash32Custom`과 동일한 알고리즘을 적용하고, 8자리 16진수 키는 그대로 trait ID로 해석한다. 각 trait ID의 `MAX(Level)`을 최대 유효 레벨로 사용한다. 이 방식으로 대미지 상한 `SKILL_020_00`은 trait ID `0xDC584F60`, 최대 레벨 `65`로 확인됐다.

재생성 명령은 다음과 같다. 게임 원본 테이블과 중간 SQLite는 저장소 밖 임시 폴더에만 둔다.

```powershell
GBFRDataTools.exe extract -i 'D:\SteamLibrary\steamapps\common\Granblue Fantasy Relink\data.i' -f 'system/table/skill_status.tbl' -o $tables
GBFRDataTools.exe tbl-to-sqlite -i (Join-Path $tables 'system\table') -o $sqlitePath -v 2.0.2
cargo run --locked --package gbfr-logs --bin build_trait_caps -- $sqlitePath 'src-tauri\assets\trait-caps.json' '63340832BCF731FBC97796F686B05C988418E83D451D4A49B2244A85D00E297F'
```
