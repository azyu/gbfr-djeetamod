# Granblue Fantasy: Relink 데미지 미터 설계

작성일: 2026-07-18

대상 게임: Granblue Fantasy: Relink Endless Ragnarok 2.0.2 (Windows/Steam)

## 1. 목표

현재 전투의 파티원별 누적 피해와 DPS를 게임 화면 좌측 파티 HUD 아래에 작은 오버레이로 표시한다. 기본 언어는 한국어다.

MVP는 다음 정보만 표시한다.

- 캐릭터 이름
- 누적 피해량
- 파티 내 최고 누적 피해를 기준으로 한 상대 바 그래프
- DPS

스킬 상세, 히트 수, 장비, 과거 로그, 차트, 네트워크 랭킹은 MVP 범위에서 제외한다.

## 2. 기술 기반

[`onelittlechildawa/gbfr-logs`](https://github.com/onelittlechildawa/gbfr-logs/tree/codex/game-2.0.2-compat) Awa Edition 1.8.6을 MIT 조건에 따라 포크한다. 이 포크의 게임 2.0.2 데미지 훅, 전투 종료 감지, 온라인 파티원 식별 및 동일 캐릭터 분리 로직을 기준 구현으로 사용한다.

구조는 다음과 같다.

```text
게임 ProcessDamageEvent
  -> src-hook 주입 DLL
  -> named pipe 이벤트
  -> Rust/Tauri 집계기
  -> TypeScript 소형 투명 오버레이
```

일반 데이터 MOD용 Reloaded-II와 `gbfrelink.utility.manager`는 MVP의 필수 의존성으로 추가하지 않는다. 인게임 D3D11/ImGui 렌더링도 MVP에서는 사용하지 않는다.

## 3. 구성요소와 책임

### 3.1 데미지 캡처

- Awa Edition의 2.0.2 시그니처와 `DamageInstance` 레이아웃을 사용해 데미지 이벤트를 수집한다.
- 각 이벤트는 공격 주체, 캐릭터 타입, 피해량, 액션 및 피해 플래그를 포함한다.
- 게임 상태를 변경하거나 입력을 자동화하거나 네트워크 패킷을 보내지 않는다.
- 핵심 데미지 훅과 보조 훅을 분리한다. 이름, 전투 종료 등 보조 훅이 실패해도 데미지 집계 프로세스가 함께 충돌하지 않아야 한다.

### 3.2 전투 집계

- 캐릭터 인스턴스별로 누적 피해를 합산한다.
- 같은 캐릭터가 파티에 둘 이상 있어도 서로 다른 인스턴스로 유지한다.
- 퀘스트 안의 여러 적에게 준 피해를 하나의 전투 결과로 합산한다.
- DPS 시간은 첫 유효 피해 이벤트에서 시작한다.
- DPS는 `누적 피해 / 첫 피해 이후 경과 시간(초)`으로 계산한다.
- 전투 종료가 확인되면 DPS 계산의 경과 시간을 즉시 동결한다. 리워드 화면 직전까지 결과를 유지하는 동안 DPS 값은 변하지 않는다.
- 표시 순서는 누적 피해 내림차순이며 최대 4명이다.
- 바 길이는 `해당 캐릭터 누적 피해 / 현재 최고 누적 피해`로 계산한다.
- 최고 누적 피해가 0이면 모든 바 길이를 0으로 처리한다.

### 3.3 전투 수명 주기

```text
대기
  -> 첫 유효 피해: 집계 시작 및 오버레이 표시
  -> 전투 중: 집계와 표시 갱신
  -> 마지막 적 처치: 최종 결과 유지
  -> 리워드 화면 진입 직전: 오버레이 숨김 및 집계 초기화
```

전투 종료 훅이 실패할 경우 리워드 화면 전환 감지 또는 다음 퀘스트 시작을 보조 초기화 조건으로 사용한다. 다음 퀘스트의 피해가 이전 결과에 더해져서는 안 된다.

### 3.4 오버레이 UI

- 1920x1080 기준 기본 크기는 330x145px이다.
- 기본 위치는 화면 좌측 파티 HUD 바로 아래다.
- 창은 투명, 테두리 없음, 항상 위로 설정한다.
- 전투 및 일반 사용 중에는 마우스 입력이 게임으로 통과한다.
- 설정 모드에서만 위치와 크기, 투명도를 변경할 수 있다.
- 해상도와 UI 배율에 맞춰 크기와 위치를 비례 조정하되 기본 배치를 유지한다.
- 250ms마다 표시를 갱신하고 바 너비에는 150ms 선형 보간을 적용한다.

각 행의 형식은 다음과 같다.

```text
[상대 누적 피해 바] 캐릭터 이름    누적 피해    DPS
```

제목, 상태 메시지 및 설정 UI의 기본 언어는 한국어다. 게임이 제공하는 한국어 캐릭터 이름을 우선 사용하고, 매핑이 없으면 내부 식별자 대신 안전한 대체 이름을 표시한다.

## 4. 오류 처리

- 게임 프로세스를 찾지 못하면 오버레이를 띄우지 않고 설정 창에 한국어 연결 상태를 표시한다.
- 필수 2.0.2 데미지 시그니처를 찾지 못하면 훅을 설치하지 않고 호환되지 않는 게임 버전임을 알린다.
- named pipe 연결이 끊기면 마지막 값을 고정하지 않고 오버레이를 숨긴다.
- 알 수 없는 캐릭터는 별도 행으로 집계하되 다른 캐릭터 데이터와 합치지 않는다.
- 잘못된 음수 피해, 비정상적으로 큰 값 또는 중복 이벤트를 조용히 보정하지 않는다. 진단 로그에 남겨 역공학 오류를 확인할 수 있게 한다.
- 훅 설치 실패와 게임 종료가 게임 프로세스 충돌로 이어지지 않아야 한다.

## 5. 검증 계획

### 자동 테스트

- 이벤트 시퀀스에서 캐릭터별 누적 피해와 DPS 계산
- 동일 캐릭터 인스턴스 분리
- 최고 피해 기준 상대 바 비율
- 누적 피해 내림차순 및 최대 4명 제한
- 첫 피해 이전과 리워드 화면 전환 시 상태 초기화
- 알 수 없는 캐릭터와 보조 훅 실패 처리
- 한국어 기본 설정과 대체 이름 표시

### 게임 내 수동 검증

- 훈련장 단일 공격의 표시 피해와 누적 피해 일치
- 4인 파티의 캐릭터별 피해 분리
- 같은 캐릭터가 두 명인 온라인 파티의 행 분리
- 추가 피해, DoT, SBA 및 링크 어택의 중복·누락 확인
- 여러 적이 등장하는 퀘스트의 전투 단위 합산
- 마지막 적 처치 뒤 결과 유지
- 리워드 화면 직전 숨김 및 초기화
- 1920x1080에서 파티 HUD, 적 HP와 상태 표시를 가리지 않는지 확인
- 게임 종료, 훅 실패 및 파이프 단절 시 게임과 오버레이가 안전하게 종료되는지 확인

현재 2.0.2에서 특정 커널 안티치트의 존재 여부는 확인된 사실로 가정하지 않는다. 온라인 검증은 별도 테스트 계정 또는 위험을 인지한 환경에서 제한적으로 수행한다.

## 6. 배포와 위험 고지

- 원본과 Awa Edition의 MIT 저작권 및 허가문을 보존한다.
- 배포물에는 지원 게임 버전 2.0.2와 바이너리 체크섬을 명시한다.
- 이 도구는 DLL 주입, 게임 메모리 읽기 및 런타임 코드 후킹을 사용한다.
- 공식 허용 또는 화이트리스트된 도구로 표현하지 않는다.
- 온라인 사용과 계정 제재 위험이 0이라고 보증하지 않는다.
- 데이터 변조, 입력 자동화, 치트 기능 및 외부 랭킹 전송은 추가하지 않는다.
