# 컴팩트 데미지 미터 4행 레이아웃 설계

## 목표

기존 330×145 컴팩트 데미지 미터 창에서 파티원 네 명을 모두 잘림 없이 표시한다. 현재 데이터 모델과 React 렌더링은 이미 최대 4행을 제공하므로, 변경 범위는 CSS 레이아웃과 회귀 테스트로 제한한다.

## 원인

헤더에 `height: 25px`와 상하 `padding: 4px`가 함께 지정되어 있지만 `box-sizing`이 적용되지 않았다. 따라서 헤더가 실제로 33px를 차지하고, 29px 높이의 네 번째 행이 145px 창 아래쪽에서 잘린다.

## 변경

- `.compact-meter__header`에 `box-sizing: border-box`를 지정해 선언된 25px 안에 패딩이 포함되게 한다.
- `.compact-meter__row` 높이를 29px에서 28px로 줄인다.
- 전체 콘텐츠 높이는 `25 + 28 × 4 = 137px`가 되어 145px 창의 테두리 안에서도 여유를 확보한다.
- 11px 행 글꼴, 12px 헤더 글꼴, 창 크기, 데이터 정렬, 최대 4행 제한은 유지한다.

## 검증

- CSS 계약 테스트로 헤더의 `box-sizing: border-box`와 28px 행 높이를 고정한다.
- 기존 컴포넌트 테스트로 문자명, 데미지, DPS 및 바 표시가 유지되는지 확인한다.
- 기존 모델 테스트로 최대 4명 제한과 정렬이 유지되는지 확인한다.
- 프런트엔드 전체 테스트, 타입 검사, 린트, 프로덕션 빌드를 실행한다.
