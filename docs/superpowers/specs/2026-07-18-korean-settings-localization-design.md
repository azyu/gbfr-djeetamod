# Djeeta MOD 설정 화면 한국어화 설계

## 목표

한국어가 선택된 상태에서 Djeeta MOD 설정 화면에 남아 있는 하드코딩 영어 문구를 자연스러운 한국어로 표시한다. 영어 등 다른 언어 선택 기능은 유지한다.

## 범위

- `Settings.tsx`에서 직접 표시되는 설정 화면 문구만 변경한다.
- 다음 영어 문구를 번역 키로 교체한다.
  - `Customize Overlay Meter Columns` → `오버레이 미터 열 설정`
  - `Add column` → `열 추가`
  - `Remove column` → `열 제거`
  - 색상 입력 placeholder `Color` → `색상`
- 기존 설정 명칭과 설명은 한국어 번역 파일의 값을 사용한다.
- 플레이어 색상 표기는 `플레이어 1`부터 `플레이어 4`까지 한국어로 통일한다.
- `DPS`, `SBA`, `DMG`, `SPS`처럼 게임 및 미터에서 사용하는 약어는 그대로 유지한다.

## 제외 범위

- 로그 목록, 피해 상세, 스킬명 및 적 이름의 전면 번역
- 내부 enum, 저장 키, IPC 메시지와 Rust 타입 이름 변경
- 설정 화면의 배치, 기능 또는 기본값 변경
- 영어 이외 언어 번역의 전면 보정

## 구현 방식

설정 화면에 한국어 문자열을 직접 하드코딩하지 않는다. 영어와 한국어 `ui.json`에 공통 번역 키를 추가하고 `Settings.tsx`는 `t(...)`를 통해 해당 값을 표시한다. 이렇게 하면 한국어 기본값과 기존 언어 전환 기능을 함께 유지할 수 있다.

추가할 설정 전용 키는 다음과 같다.

- `ui.color-placeholder`
- `ui.customize-overlay-columns`
- `ui.add-column`
- `ui.remove-column`

기존 번역 키인 플레이어 색상, 디버그 모드, 미터 열 이름 및 설명은 한국어 값을 보정해 재사용한다.

## 검증

- 설정 화면 소스에 대상 영어 하드코딩 문자열이 남아 있지 않아야 한다.
- 영어와 한국어 번역 파일에 새 키가 모두 존재해야 한다.
- 한국어 값은 각각 `색상`, `오버레이 미터 열 설정`, `열 추가`, `열 제거`여야 한다.
- `DPS`, `SBA`, `DMG`, `SPS` 약어는 변경하지 않아야 한다.
- 프런트엔드 테스트, 타입 검사, 린트 및 포맷 검사가 통과해야 한다.
