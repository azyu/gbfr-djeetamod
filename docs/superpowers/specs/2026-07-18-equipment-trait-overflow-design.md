# Djeeta MOD 장비 특성 초과 분석 설계

작성일: 2026-07-18
대상: Granblue Fantasy: Relink - Endless Ragnarok 2.0.2, Windows x64

## 1. 목표

MOD 프로그램에서 모든 로컬 캐릭터의 장비 특성을 확인하고, 동일 특성의 합산 레벨이 2.0.2 최대 유효 레벨에 도달했거나 초과했는지 보여준다.

예시:

```text
대미지 상한  72 / 65  · 7 초과
공격력       50 / 50  · 최대
퀵 어빌리티  30 / 45  · 정상
신규 특성    15 / ?   · 최대치 미확인
```

이 기능은 적이나 전투 상황에 따른 실제 대미지 상한을 계산하지 않는다. 장비 특성 레벨의 유효 최대치와 초과분만 분석한다.

## 2. 범위

### 포함

- 로컬 저장 데이터에서 사용할 수 있는 모든 캐릭터
- 진 12칸의 주 특성과 보조 특성
- 무기의 일반 trait ID/level
- 가호석의 일반 trait ID/level
- Master Traits와 Summon traits 중 게임 장비 화면의 동일 trait level 합계에 직접 참여하는 항목
- 특성별 현재 합계, 최대 유효 레벨, 초과 레벨
- 특성 합계에 기여한 장비별 내역
- 2.0.2에서 검증된 특성 최대치 카탈로그
- 한국어 특성명과 미확인 상태

### 제외

- 온라인 파티원의 장비
- 장비 메뉴에서 현재 선택된 캐릭터 자동 추적
- 적 방어력, 속성, 치명타, 버프·디버프를 포함한 대미지 계산
- 공격 스킬별 실제 대미지 상한
- 장비, 세이브 또는 게임 메모리 변경
- 최대치가 확인되지 않은 특성의 추정 판정

## 3. 접근 방식

2.0.2 게임 데이터에서 만든 버전 고정 특성 최대치 카탈로그와 로컬 캐릭터 장비 스냅샷을 결합한다. MOD 창은 장비 메뉴가 열려 있지 않아도 캐릭터별 결과를 보여준다.

게임 UI 내부 집계 결과를 직접 훅하는 방식은 초기 범위에서 사용하지 않는다. UI 레이아웃 의존성을 피하고 모든 로컬 캐릭터를 한 화면에서 선택할 수 있게 하기 위해서다. 게임 장비 화면의 총 특성 레벨은 테스트 기준으로만 사용한다.

## 4. 아키텍처

```text
게임 2.0.2 로컬 캐릭터/장비 레코드
  -> src-hook: 검증된 장비 스냅샷 캡처
  -> protocol: append-only 로컬 장비 메시지
  -> src-tauri: 캐릭터별 최신 스냅샷 보관
  -> trait analyzer: 동일 trait 합산 및 최대치 비교
  -> Tauri event/command
  -> React 장비 분석 화면
```

### 4.1 훅

훅은 2.0.2 로컬 캐릭터 목록 또는 장비 갱신 경계를 찾아 읽기 전용 스냅샷을 만든다. 스냅샷은 캐릭터 ID와 각 특성 원천을 보존한다.

장비 읽기와 검증은 플레이어 신원 훅과 분리한다. 장비 레이아웃 실패가 기존 대미지 미터의 신원·대미지 캡처를 중단시키지 않게 한다. 다만 장비 분석 화면은 부분 합계를 정상 결과로 표시하지 않는다.

### 4.2 프로토콜

기존 `PlayerLoadEvent`는 진뿐 아니라 무기, 오버마스터리, 스탯을 필수로 요구한다. 2.0.2에서 이 전체 레이아웃을 함께 검증할 수 있으면 기존 이벤트를 재사용한다.

전체 레이아웃이 검증되지 않고 장비 특성 원천만 안전하게 읽을 수 있으면, 장비 분석 전용 메시지를 `Message` enum의 마지막에 추가한다. 기존 bincode variant는 재정렬하지 않는다. 전용 메시지는 다음 의미를 갖는다.

- 로컬 캐릭터 식별자
- 완전한 스냅샷 여부
- 특성 원천 목록
- 각 원천의 종류, 슬롯, 항목 ID, trait ID, trait level
- 캡처가 지원되지 않을 때의 명시적인 상태

### 4.3 특성 최대치 카탈로그

카탈로그는 `trait ID → max effective level`의 2.0.2 버전 고정 데이터다. 생성 또는 검증에 사용한 실행 파일·자산 SHA-256과 출처를 함께 기록한다.

최대치가 없는 고유 단일 효과는 게임 데이터가 명시한 유효 레벨을 사용한다. 최대치를 검증할 수 없는 특성은 카탈로그에서 값을 만들지 않고 `미확인`으로 둔다.

### 4.4 분석기

분석기는 UI와 독립된 순수 로직으로 구현한다.

1. 스냅샷이 완전하고 지원되는지 확인한다.
2. 모든 특성 원천을 trait ID로 그룹화한다.
3. 각 그룹의 레벨을 정수로 합산한다.
4. 카탈로그의 최대치와 비교한다.
5. 상태와 초과분을 만든다.

판정 규칙:

```text
max 없음                  -> unknown
total < max               -> below
total == max              -> capped
total > max               -> overflow, overflow = total - max
```

빈 슬롯과 trait ID 0은 합산하지 않는다. 같은 진의 주·보조 특성이 같은 ID라면 두 레벨을 각각 기여분으로 합산한다. 원천 목록은 상세 UI를 위해 유지한다.

### 4.5 백엔드 상태

Tauri 백엔드는 캐릭터 ID별 최신 완전 스냅샷을 보관한다. 장비 변경을 감지하면 해당 캐릭터만 교체하고 React에 갱신 이벤트를 보낸다. 전투 encounter와 장비 분석 상태는 별개로 유지한다.

저장 로그의 기존 CBOR encounter 형식은 변경하지 않는다. 장비 분석은 라이브 로컬 장비 도구이며 과거 전투 로그 마이그레이션을 요구하지 않는다.

### 4.6 React UI

기존 앱 내비게이션에 `장비 분석` 화면을 추가한다.

- 상단: 로컬 캐릭터 선택
- 본문: 특성명, 현재/최대, 상태, 초과분
- 상세 확장: 진 슬롯, 무기, 가호석 및 다른 검증된 원천별 기여 레벨
- 빈 상태: 게임 미연결, 지원되지 않는 레이아웃, 장비 데이터 대기 상태를 구분

정렬 순서는 `overflow → capped → below → unknown`이다. 같은 상태에서는 한국어 특성명으로 정렬한다. 초과분은 색상과 텍스트를 함께 사용해 색상만으로 의미를 전달하지 않는다.

## 5. 완전성 및 오류 처리

장비 분석은 완전한 입력만 정상 결과로 표시한다.

- 지원 2.0.2 해시 또는 검증된 시그니처가 아니면 장비 캡처를 설치하지 않는다.
- 시그니처는 유일하게 한 번만 매치해야 한다.
- 캐릭터 ID, 슬롯 인덱스, 특성 ID와 레벨을 범위 검사한다.
- 진 슬롯 수는 12 이하이어야 한다.
- 일반 trait level 합계에 참여하는 필수 원천이 미수집이면 캐릭터 전체를 `장비 정보 미지원`으로 표시한다.
- 포인터 읽기 실패를 빈 슬롯이나 0레벨로 변환하지 않는다.
- 미확인 최대치를 임의의 공통 최대치로 대체하지 않는다.
- 장비 캡처 실패는 기존 미터 연결을 `Ready`에서 `Unsupported`로 바꾸지 않는다. 장비 분석 기능만 독립적으로 미지원 상태가 된다.
- 기존 `HookStatus::Unsupported` latch 동작은 보존한다.

## 6. 갱신 수명 주기

- 게임 연결 후 로컬 캐릭터 장비 레코드가 준비되면 초기 스냅샷을 발행한다.
- 장비 변경 경계가 호출되면 변경된 캐릭터의 스냅샷을 다시 발행한다.
- 동일한 스냅샷은 중복 발행하지 않는다.
- 게임 연결 해제 또는 세이브 전환 시 모든 장비 분석 상태를 지운다.
- 캐릭터 선택은 React 상태이며 훅 캡처 범위에 영향을 주지 않는다.

## 7. 테스트 전략

### 훅 단위 테스트

- 진 12칸과 빈 슬롯 디코딩
- 주·보조 특성 디코딩
- 무기와 가호석 특성 디코딩
- 유효하지 않은 포인터, 슬롯 수, ID, 레벨 거부
- 동일 스냅샷 중복 억제
- 불완전한 필수 원천의 미지원 판정

실제 오프셋과 시그니처는 순수 단위 테스트가 아니라 2.0.2 수동 테스트에서 검증한다.

### 프로토콜 테스트

- 장비 메시지 bincode 왕복
- 기존 variant ordinal 불변
- legacy damage wire 디코딩 유지

### 분석기 단위 테스트

- `64/65 → below`
- `65/65 → capped`
- `72/65 → overflow +7`
- 같은 특성이 여러 진·무기에 분산된 합산
- 같은 진의 주·보조 특성이 같은 경우의 합산
- trait ID 0과 빈 슬롯 제외
- 최대치 미확인 특성
- 불완전 스냅샷 거부

### React 테스트

- 캐릭터 선택과 스냅샷 교체
- overflow/capped/below/unknown 정렬
- `현재/최대` 및 초과 텍스트
- 원천별 상세 기여 표시
- 게임 미연결과 장비 미지원 상태 구분

### 실제 게임 2.0.2 테스트

- 게임 장비 화면의 총 특성 레벨과 모든 MOD 합계 비교
- 진 하나 교체 후 해당 캐릭터만 갱신
- V+ 주·보조 특성
- 같은 특성의 다중 진 조합과 최대치 초과
- 무기와 가호석 기여
- Master Traits와 Summons의 일반 trait level 참여 여부
- 캐릭터를 여러 번 바꾼 뒤 스냅샷 일관성
- 게임 재연결과 세이브 전환
- 잘못된 시그니처에서 게임 충돌 없이 장비 분석만 미지원 처리

## 8. 성공 기준

- MOD 창에서 모든 로컬 캐릭터를 선택할 수 있다.
- 선택한 캐릭터의 완전한 장비 특성 합계를 표시한다.
- 2.0.2에서 검증된 최대치에 대해 below/capped/overflow를 정확히 판정한다.
- `72/65 · 7 초과`와 같은 결과 및 원천별 기여를 확인할 수 있다.
- 최대치 또는 장비 원천이 불확실한 경우 정상 결과를 제조하지 않는다.
- 기존 대미지 미터, 프로토콜 호환성, 저장 로그를 깨뜨리지 않는다.
- 자동 테스트가 통과하고 게임 장비 화면과 수동 대조가 일치한다.
- `docs/testing/game-2.0.2-smoke-test.md`가 완료되기 전에는 2.0.2 호환성을 주장하지 않는다.
