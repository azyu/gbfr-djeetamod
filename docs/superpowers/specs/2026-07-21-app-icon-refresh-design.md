# Djeeta MOD 앱 아이콘 교체 설계

## 목표

사용자가 제공한 `image.png`를 얼굴 중심으로 확대해 Djeeta MOD의 앱 및 배포 아이콘으로 사용한다. 작은 작업표시줄·트레이 아이콘에서도 얼굴과 고글이 식별되어야 하며, 큰 아이콘에서는 머리와 목 장식이 자연스럽게 보여야 한다.

## 원본과 크롭

- 입력 원본은 1254×1254 RGBA PNG다.
- 얼굴, 고글, 머리카락, 목 장식이 남는 정사각형 균형 크롭을 사용한다.
- 모든 출력 크기는 동일한 크롭 원본에서 생성해 크기별 인상이 달라지지 않게 한다.
- 원본의 흰 배경과 화풍은 유지하며 AI 보정이나 재생성은 하지 않는다.

## 출력 범위

현재 `src-tauri/icons`의 Tauri 아이콘 세트를 새 원본에서 다시 생성한다.

- 앱 번들 PNG: 32×32, 128×128, 256×256
- Windows 실행 파일 및 바로가기용 ICO
- macOS 번들 설정 호환용 ICNS
- Windows 타일·스토어용 PNG 자산
- 시스템 트레이가 참조하는 32×32 PNG

`src-tauri/tauri.conf.json`의 NSIS 설정에도 같은 ICO를 설치 프로그램 아이콘으로 명시한다. 이에 따라 앱 실행 파일, 작업표시줄, 시작 메뉴·바탕 화면 바로가기, 트레이 및 설치 프로그램이 같은 시각 자산을 사용한다.

## 변경 제한

- 앱 동작, 창 구성, 훅 및 프로토콜은 변경하지 않는다.
- 기존의 미커밋된 패키징 해시 문서 변경은 보존한다.
- 사용자 파일인 `logs.db`는 읽거나 커밋하지 않는다.

## 검증

1. 생성된 필수 아이콘 파일의 존재와 이미지 크기를 검사한다.
2. ICO에 Windows용 다중 해상도 프레임이 포함되는지 확인한다.
3. `tauri.conf.json`이 유효하고 번들 설정이 새 아이콘을 참조하는지 확인한다.
4. 프론트엔드 빌드와 Tauri NSIS 패키징을 실행한다.
5. 패키징 스크립트가 갱신한 설치 파일·훅 SHA-256을 문서에 반영한 상태로 보존한다.
