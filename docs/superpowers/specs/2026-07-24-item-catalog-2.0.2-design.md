# 2.0.2 아이템 번역 카탈로그 설계

## 목표

Djeeta MOD가 사용하는 한국어·영어 아이템 번역 카탈로그를 Granblue Fantasy:
Relink Endless Ragnarok 2.0.2 게임 자산 기준으로 전체 재생성한다. 아이템 분석
화면을 포함한 모든 기존 아이템 이름 소비자가 구버전 1.3 계열 카탈로그나 16진수
ID 폴백에 의존하지 않게 한다.

## 범위

포함 범위는 다음과 같다.

- `src-tauri/lang/ko/items.json` 전체 재생성
- `src-tauri/lang/en/items.json` 전체 재생성
- 2.0.2 `item.tbl`의 아이템 ID와 번역 키 결합
- 2.0.2 한국어·영어 메시지 데이터의 아이템 이름 결합
- 결정적 출력과 카탈로그 완전성을 검증하는 자동 테스트
- 재생성 절차와 정확한 게임 버전 기록
- 기존 아이템 분석 실게임 체크리스트에 남은 작업 추가

다음은 포함하지 않는다.

- 한국어·영어 이외 언어의 `items.json` 업데이트
- 원본 게임 아카이브나 추출된 저작권 자산의 저장소 포함
- 아이템 설명, 획득처, 보유 한도 또는 분류를 번역 JSON에 추가
- 게임 데이터 추출 도구의 저장소 내 재구현
- 미완료 실게임 검증을 자동 테스트 결과로 통과 처리

## 입력과 출처

생성 입력은 사용자가 소유한 Granblue Fantasy: Relink Endless Ragnarok 2.0.2
설치본에서 추출한 다음 데이터다.

1. 2.0.2 형식으로 변환한 `item.tbl`
2. 한국어 아이템 이름 메시지 데이터
3. 영어 아이템 이름 메시지 데이터

생성 절차는 고정된 게임 버전 `2.0.2`와 지원 실행 파일 SHA-256
`63340832BCF731FBC97796F686B05C988418E83D451D4A49B2244A85D00E297F`를 문서에
기록한다. 원본 및 중간 추출물은 임시 작업 위치에만 두고 커밋하지 않는다.

## 생성 방식

`src-tauri/examples/build_item_catalog.rs`에 유지보수용 생성기를 둔다. 생성기는
변환된 SQLite 데이터베이스, 한국어 `text.msg`, 영어 `text.msg`, 두 출력 경로,
지원 실행 파일 SHA-256을 명시적 인자로 받는다. 기존 `build_trait_caps` 예제의
읽기 전용 SQLite 및 MessagePack 파서를 재사용할 수 있는 범위에서 같은 입력
계약을 따른다. 생성기는 다음 순서로 동작한다.

1. 변환된 아이템 테이블에서 아이템 ID와 번역 키를 읽는다.
2. 같은 번역 키를 한국어와 영어 메시지 데이터에서 각각 찾는다.
3. 아이템 ID를 게임이 사용하는 8자리 소문자 16진수 해시 키로 변환한다.
4. 각 언어의 값을 기존 계약인 `{ "key": string, "text": string }` 형태로 만든다.
5. 해시 키 오름차순으로 정렬해 결정적인 JSON을 출력한다.

생성기는 다음 조건에서 출력을 거부한다.

- 중복 아이템 ID 또는 해시 충돌
- 빈 번역 키나 빈 이름
- 한 언어에만 존재하는 아이템
- 한국어·영어에서 같은 아이템 ID가 서로 다른 번역 키를 가리키는 경우
- 입력에서 파악한 전체 아이템 수와 출력 수가 다른 경우

기존 런타임 계약은 변경하지 않는다. `translateItemId`와 i18next 네임스페이스는
재생성된 파일을 그대로 소비한다.

## 검증

자동 테스트는 다음을 증명한다.

- 한국어·영어 카탈로그의 해시 키 집합과 번역 키가 일치한다.
- 모든 항목의 해시 키, 번역 키, 번역문이 유효하다.
- 2.0.2에서 추가된 대표 아이템이 양쪽 언어에 존재한다.
- `src-tauri/data/ordinary-items-2.0.2.json`의 일반 아이템 281개가 양쪽 언어에
  모두 존재한다.
- 생성기를 같은 입력으로 두 번 실행하면 바이트 단위로 같은 결과가 나온다.
- 기존 아이템 이름 소비자와 아이템 분석 화면의 회귀 테스트가 통과한다.

일반 프런트엔드 검사와 Rust 워크스페이스 검사는 프로젝트의 기존 필수 검증 명령을
그대로 따른다.

## 후속 작업 기록

`docs/testing/game-2.0.2-item-analysis-probe.md`에 `Remaining tasks` 체크리스트를
추가하고 다음 작업을 관리한다.

- 한국어·영어 아이템 번역 카탈로그 2.0.2 재생성 및 자동 검증
- 한 아이템의 통제된 수량 증가 검증
- 한 아이템의 통제된 수량 감소 검증
- 899, 900, 999 경계값 실게임 검증
- 게임 내 아이템 메뉴 정렬·필터 변경 안정성 검증
- 게임 재시작 후 로케이터 재검증 3회
- 필수 프런트엔드·Rust 회귀 검사와 빌드

완료된 자동 작업만 체크하고, 게임 조작이나 재시작이 필요한 행은 실제로 관찰한
증거가 있을 때만 체크한다.

## 오류 및 안전성

- 잘못되거나 불완전한 입력으로 기존 카탈로그를 부분 덮어쓰지 않는다.
- 한국어와 영어 출력은 모두 검증된 뒤 함께 교체한다.
- 생성 실패 시 기존 `items.json` 두 파일을 유지한다.
- `logs.db`와 게임 저장 데이터는 읽거나 수정하지 않는다.
- 번역 갱신만으로 게임 2.0.2 전체 호환성을 주장하지 않는다.

## 완료 조건

- 한국어·영어 `items.json`이 2.0.2 자산의 전체 아이템 집합과 일치한다.
- 두 언어의 ID 및 번역 키 집합이 정확히 같다.
- 일반 아이템 281개가 모두 이름으로 표시될 수 있다.
- 재생성 절차가 문서화되고 결정적으로 반복 가능하다.
- 체크리스트에 자동 완료 항목과 남은 수동 검증이 명확히 구분된다.
- 관련 집중 테스트와 프로젝트 필수 전체 검증이 통과한다.
