# 미터 창 및 관리 메뉴 설계

## 목표

Djeeta MOD의 작은 실시간 미터와 관리 창의 역할을 명확히 분리한다. 미터는 게임 위에 표시되는 오버레이로 유지하고, 관리 창 왼쪽 메뉴에서 미터 표시 여부를 즉시 제어한다.

## 창 역할

### 미터 창

- 게임 위에 표시되는 실시간 대미지 오버레이다.
- `always on top`을 항상 적용한다.
- Windows 작업 표시줄에는 표시하지 않는다.
- 전투 중이 아니어도 창과 드래그 가능한 헤더를 표시한다.
- 헤더의 Tauri drag region을 이용해 마우스로 위치를 옮긴다.
- 기본 상태는 표시됨(ON)이다.

### 관리 창

- 창 제목은 `Djeeta MOD`로 표시한다.
- Windows 작업 표시줄에 표시되는 유일한 Djeeta MOD 창이다.
- 기존 `Djeeta MOD - Logs` 창의 전투 기록, 진 분석, 설정 기능을 유지한다.

## 왼쪽 메뉴

위에서 아래로 다음 항목을 표시한다.

1. `데미지 미터`와 오른쪽 ON/OFF 스위치
2. `진 특성 상한 분석`
3. `전투 기록`
4. `설정`

미터 항목은 페이지 이동 링크가 아니다. 스위치와 행 클릭 모두 같은 표시 상태를 전환한다. 나머지 세 항목은 기존 화면으로 이동한다.

## 미터 ON/OFF 상태

- ON이면 미터 창을 즉시 표시하고 항상 위 상태를 유지한다.
- OFF이면 미터 창을 즉시 숨긴다.
- 상태는 기존 Zustand 영속 설정에 저장해 앱 재실행 후에도 복원한다.
- 저장값이 없는 기존 사용자와 신규 사용자의 기본값은 ON이다.
- 관리 창이 시작될 때 저장 상태를 Tauri 명령에 동기화해 초기 창 가시성을 확정한다.
- 미터 창의 작업 표시줄 항목은 상태와 관계없이 생성하지 않는다.

## 드래그와 클릭 통과

- 미터 헤더의 기존 `data-tauri-drag-region`을 유지해 드래그 이동을 지원한다.
- 클릭 통과의 런타임 기본값은 OFF로 변경한다. 따라서 새 실행 직후 헤더를 드래그할 수 있다.
- 사용자가 위치를 잡은 뒤 시스템 트레이의 `클릭 통과`를 켤 수 있다.
- 클릭 통과가 켜진 동안에는 Windows가 미터 창에 마우스 입력을 전달하지 않으므로 드래그할 수 없다.
- 기존 시스템 트레이의 `항상 위` 토글은 제거한다. 미터의 항상 위 동작은 제품 불변 조건이다.

## 백엔드 명령

토글이 아니라 명시적 상태를 받는 `set_meter_enabled(enabled: bool)` 명령을 추가한다. 명령은 `main` 창에 다음을 적용한다.

- `enabled = true`: 창 표시 및 always-on-top 적용
- `enabled = false`: 창 숨김

명시적 setter를 사용해 프런트엔드 재마운트나 중복 이벤트가 상태를 반대로 뒤집는 문제를 피한다. 프런트엔드의 영속 설정을 사용자 선택의 출처로 사용하며 별도 DB 설정 테이블은 만들지 않는다.

## 라벨과 다국어

- 기존 하드코딩된 `Logs`와 `Settings` 메뉴 라벨을 i18n 키로 교체한다.
- 한국어는 `데미지 미터`, `진 특성 상한 분석`, `전투 기록`, `설정`을 사용한다.
- 영어는 `Damage Meter`, `Sigil Trait Cap Analysis`, `Battle Records`, `Settings`를 사용한다.

## 오류 처리

- 미터 창을 찾지 못하거나 show/hide 호출이 실패하면 Tauri 명령 오류로 프런트엔드에 반환한다.
- 명령이 실패하면 저장된 스위치 값을 성공한 상태로 확정하지 않고 기존 값으로 복원한다.
- 앱 시작 동기화 실패는 관리 창 전체 렌더링을 막지 않는다.

## 검증

자동 테스트는 다음을 보장한다.

- 미터 설정의 기본값이 ON이다.
- 관리 메뉴에 미터 스위치, 진 분석, 전투 기록, 설정이 표시된다.
- 미터 스위치가 명시적 ON/OFF 명령을 호출하고 성공 시 저장 상태를 갱신한다.
- 명령 실패 시 스위치 상태가 복원된다.
- 한국어와 영어 메뉴 라벨이 모두 존재한다.
- Tauri 설정에서 미터 창은 always-on-top이고 작업 표시줄에서 숨겨진다.
- 백엔드 명령은 ON일 때 show, OFF일 때 hide를 선택한다.

구현 완료 후 다음 검증을 모두 실행한다.

1. `npm run format-check`
2. `npm run lint`
3. `npm run tsc`
4. `npm test -- --run`
5. `npm run build`
6. `cargo build --release --locked --package hook`
7. `cargo test --workspace --all-targets --locked`

마지막으로 실제 게임에서 미터가 작업 표시줄에 나타나지 않는지, 항상 위에 유지되는지, 헤더 드래그와 관리 메뉴 ON/OFF가 동작하는지 수동 확인한다.
